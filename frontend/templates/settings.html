Paso 1 · Vincular cuenta de Google

En tu workstation con browser, instala rclone, ejecuta rclone authorize "drive", autoriza con la cuenta corporativa donde se subirán los backups, y pega el JSON resultante abajo. rclone se encarga de rotar el token automáticamente — no tienes que repetirlo.

¿No tienes rclone? Instalación rápida
Windows (PowerShell, requiere winget)
winget install Rclone.Rclone
Alternativa: descargar el zip desde rclone.org/downloads y agregar la carpeta al PATH.
Linux / macOS
curl https://rclone.org/install.sh | sudo bash
O vía gestor de paquetes: apt install rclone · brew install rclone · dnf install rclone.
Una vez instalado, en una terminal con browser disponible corre rclone authorize "drive". Se abrirá Google → autoriza → rclone imprimirá el JSON del token. Cópialo entero (incluye las llaves { }) y pégalo abajo.

Paso 2 · Dónde guardar los archivos

Rutas a respaldar

Una ruta absoluta por línea (o separadas por espacios). Se empaquetan juntas en cada archivo .tar.zst. Respeta la lista de exclusiones de /opt/snapshot-V3/core/etc/excludes.list.

Backup mensual (taxonomía y encriptación)

El día 1 de cada mes (02:00 UTC) se genera un .tar.zst con el contenido de las rutas configuradas arriba y se sube al shared Drive en la ruta taxonómica que defines aquí.

Programación de los backups

Cada tipo de backup tiene su propio timer en systemd. Mensual del sistema es el archive cold-storage de las rutas configuradas arriba; Bases de datos hace dump de cada engine que tengas activado abajo. Son independientes — podés tener uno sin el otro.

Vinculación con servidor central

Cuando se configura, cada backup envía un heartbeat al central con metadata (host, tamaño, status). El central usa esos heartbeats para alertas no_heartbeat, backup_shrink, etc. Sin esto, el cliente respalda igual a Drive — solo que el central no se entera. Pídele al admin del central que cree un Cliente con tu proyecto y te emita un token (/dashboard-central/clients).
